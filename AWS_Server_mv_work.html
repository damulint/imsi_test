Amazon Lightsail Amazon Linux 2 → Amazon Linux 2023 이전 작업 가이드

대상 환경: nginx + PHP-FPM 8.0.30 + MariaDB + 아미나빌더/그누보드

웹 루트: /home/계정/www

작성일: 2026-05-21 / 목적: 신규 AL2023 서버 구축 후 파일·DB 이전 및 DNS 또는 Static IP 전환

0. 핵심 요약

권장 방식기존 Amazon Linux 2 인스턴스를 직접 업그레이드하지 않고, 새 Amazon Linux 2023 Lightsail 인스턴스를 생성한 뒤 소스·DB·설정을 이전한다.

현재 PHP기존 PHP 버전이 8.0.30이므로 새 서버도 우선 PHP 8.0 계열 또는 가장 가까운 8.x 계열로 맞춘 뒤 테스트한다.

웹 루트기존 웹 루트는 /home/계정/www 기준으로 작성한다. 실제 계정명은 이 문서에서 WEBUSER로 표기한다.

중요: 아미나빌더/그누보드/스킨/플러그인 호환성 때문에 OS 이전, PHP 업그레이드, 그누보드 업데이트, 아미나빌더 업데이트를 동시에 진행하지 않는다. 이번 작업은 우선 OS와 서버 이전에 집중한다.

1. 작업 변수 정리

아래 값은 실제 환경에 맞게 바꿔서 사용한다.

항목예시설명
WEBUSERec2-user 또는 사이트 계정명웹 루트가 /home/WEBUSER/www인 계정
DOMAINexample.com운영 도메인
DB_NAMEgnuboard그누보드/아미나 DB명
DB_USERgnuboard_userDB 접속 사용자
NEW_IP새 Lightsail 공인 IP신규 AL2023 서버 IP
OLD_IP기존 Lightsail 공인 IP기존 AL2 서버 IP
 
# 이 문서의 예시 변수
WEBUSER="ec2-user"
DOMAIN="example.com"
DB_NAME="gnuboard"
DB_USER="gnuboard_user"
WEBROOT="/home/${WEBUSER}/www"

2-3. DB 정보 확인

3. 기존 서버 백업

필수: 작업 전 Lightsail 콘솔에서 기존 인스턴스의 수동 스냅샷을 생성한다. 스냅샷 완료 전에는 본 작업을 진행하지 않는다.

3-1. 웹 파일 백업

sudo mkdir -p /home/계정/migration-backup
cd /home/계정
sudo tar czvpf /home/계정/migration-backup/www-backup-$(date +%F).tar.gz www

3-2. DB 백업

DB명은 dbconfig.php에서 확인한 값을 사용한다.

mysqldump -u root -p \
  --single-transaction \
  --routines \
  --triggers \
  --events \
  --default-character-set=utf8mb4 \
  DB_NAME > /home/계정/migration-backup/DB_NAME-$(date +%F).sql

3-3. nginx/PHP/MariaDB 설정 백업

sudo tar czvpf /home/계정/migration-backup/etc-nginx-$(date +%F).tar.gz /etc/nginx
sudo tar czvpf /home/계정/migration-backup/etc-php-$(date +%F).tar.gz /etc/php.ini /etc/php-fpm.d 2>/dev/null || true
sudo tar czvpf /home/계정/migration-backup/etc-mysql-$(date +%F).tar.gz /etc/my.cnf /etc/my.cnf.d 2>/dev/null || true

4. 새 Amazon Linux 2023 Lightsail 인스턴스 생성

  1. Lightsail 콘솔에서 새 인스턴스를 생성한다.
  2. Blueprint는 OS Only → Amazon Linux 2023을 선택한다.
  3. 리전은 기존 서버와 동일하게 선택한다.
  4. 플랜은 기존과 동일하거나 한 단계 높은 사양을 권장한다.
  5. Networking에서 TCP 22, 80, 443을 허용한다.

4-1. 새 서버 기본 업데이트

sudo dnf update -y
sudo dnf install -y vim git unzip tar rsync wget curl policycoreutils-python-utils
sudo timedatectl set-timezone Asia/Hong_Kong

5. 새 AL2023 서버에 nginx / PHP-FPM / MariaDB 설치

5-1. nginx 설치

sudo dnf install -y nginx
sudo systemctl enable --now nginx
sudo systemctl status nginx --no-pager

5-2. PHP-FPM 설치

기존 PHP가 8.0.30이므로 새 서버에서도 우선 PHP 8.x 계열로 맞춘다. AL2023 기본 저장소에서 제공되는 PHP 버전을 먼저 확인한다.

sudo dnf module list php || true
sudo dnf search php-fpm
sudo dnf list available "php*" | head -100

기본 저장소 PHP를 사용하는 경우:

sudo dnf install -y \
  php php-fpm php-cli php-common php-mysqlnd \
  php-gd php-mbstring php-xml php-opcache \
  php-curl php-zip php-intl

php -v
php -m
참고: AL2023 기본 저장소에서 PHP 8.0이 제공되지 않고 8.1/8.2 이상만 제공될 수 있다. 이 경우 아미나/그누보드/스킨 호환성 테스트가 필수다. 반드시 전환 전에 hosts 파일 테스트로 오류 로그를 확인한다.

5-3. PHP-FPM pool 설정

sudo vi /etc/php-fpm.d/www.conf
; 주요 값 예시
user = nginx
group = nginx
listen = /run/php-fpm/www.sock
listen.owner = nginx
listen.group = nginx
listen.mode = 0660

5-4. php.ini 설정

sudo vi /etc/php.ini
date.timezone = Asia/Hong_Kong
memory_limit = 256M
upload_max_filesize = 64M
post_max_size = 64M
max_execution_time = 60
max_input_vars = 3000
short_open_tag = On
sudo systemctl enable --now php-fpm
sudo systemctl restart php-fpm
sudo systemctl status php-fpm --no-pager

5-5. MariaDB 설치

sudo dnf search mariadb
sudo dnf list available "mariadb*" | head -100
# 가능한 경우 MariaDB 10.5 계열
sudo dnf install -y mariadb105-server mariadb105 || sudo dnf install -y mariadb-server mariadb

sudo systemctl enable --now mariadb
sudo systemctl status mariadb --no-pager
sudo mysql_secure_installation

5-6. MariaDB 문자셋 설정

sudo vi /etc/my.cnf.d/mariadb-server.cnf
[mysqld]
character-set-server = utf8mb4
collation-server = utf8mb4_unicode_ci
max_allowed_packet = 64M
sudo systemctl restart mariadb

6. 소스와 DB 복원

6-1. 새 서버에 웹 계정 생성

기존 웹 루트가 /home/계정/www이므로 새 서버에도 동일한 계정명을 만드는 것을 권장한다.

sudo useradd -m 계정 || true
sudo passwd 계정
sudo mkdir -p /home/계정/www

6-2. 기존 서버에서 새 서버로 파일 전송

기존 서버에서 실행:

rsync -avz --numeric-ids \
  -e "ssh -i /path/to/key.pem" \
  /home/계정/www/ ec2-user@NEW_IP:/tmp/www/

새 서버에서 실행:

sudo rsync -av /tmp/www/ /home/계정/www/
sudo chown -R nginx:nginx /home/계정/www
sudo find /home/계정/www -type d -exec chmod 755 {} \;
sudo find /home/계정/www -type f -exec chmod 644 {} \;

# 그누보드/아미나 data 디렉터리는 쓰기 권한 필요
sudo chown -R nginx:nginx /home/계정/www/data
sudo chmod -R 707 /home/계정/www/data

6-3. DB 생성

sudo mysql -u root -p
CREATE DATABASE DB_NAME DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'DB_USER'@'localhost' IDENTIFIED BY '강력한비밀번호';
GRANT ALL PRIVILEGES ON DB_NAME.* TO 'DB_USER'@'localhost';
FLUSH PRIVILEGES;
EXIT;

6-4. DB 복원

기존 서버에서 DB 백업 파일을 새 서버로 전송:

scp -i /path/to/key.pem /home/계정/migration-backup/DB_NAME-YYYY-MM-DD.sql ec2-user@NEW_IP:/tmp/

새 서버에서 복원:

mysql -u DB_USER -p DB_NAME < /tmp/DB_NAME-YYYY-MM-DD.sql
mysql -u DB_USER -p -e "USE DB_NAME; SHOW TABLES;"

6-5. dbconfig.php 확인

sudo vi /home/계정/www/dbconfig.php
define('G5_MYSQL_HOST', 'localhost');
define('G5_MYSQL_USER', 'DB_USER');
define('G5_MYSQL_PASSWORD', 'DB비밀번호');
define('G5_MYSQL_DB', 'DB_NAME');

6-6. 캐시/세션 정리

sudo rm -rf /home/계정/www/data/cache/*
sudo rm -rf /home/계정/www/data/session/*
sudo chown -R nginx:nginx /home/계정/www/data

7. nginx 설정

아래 파일을 새로 생성한다.

sudo vi /etc/nginx/conf.d/DOMAIN.conf
server {
    listen 80;
    server_name DOMAIN www.DOMAIN;

    root /home/계정/www;
    index index.php index.html index.htm;
    charset utf-8;

    access_log /var/log/nginx/DOMAIN.access.log;
    error_log  /var/log/nginx/DOMAIN.error.log;

    client_max_body_size 64M;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location ~ \.php$ {
        try_files $uri =404;
        fastcgi_pass unix:/run/php-fpm/www.sock;
        fastcgi_index index.php;
        include fastcgi_params;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        fastcgi_param SCRIPT_NAME $fastcgi_script_name;
    }

    # 그누보드/아미나 보안 차단: 업로드/스킨/플러그인 내 PHP 직접 실행 제한
    location ~* /(data|install|plugin|skin|theme|thema|extend)/.*\.(php|php3|php4|php5|phtml)$ {
        deny all;
    }

    # 민감 파일 차단
    location ~* /(?:\.ht|\.git|\.svn|composer\.json|composer\.lock) {
        deny all;
    }

    location ~* /(config\.php|dbconfig\.php)$ {
        deny all;
    }
}

7-1. nginx 설정 검사 및 재시작

sudo nginx -t
sudo systemctl restart nginx
sudo systemctl restart php-fpm

7-2. SELinux 권한 처리

웹 루트가 /home/계정/www이므로 SELinux 때문에 403 또는 Permission denied가 날 수 있다.

sudo setsebool -P httpd_can_network_connect 1
sudo semanage fcontext -a -t httpd_sys_content_t "/home/계정/www(/.*)?"
sudo semanage fcontext -a -t httpd_sys_rw_content_t "/home/계정/www/data(/.*)?"
sudo restorecon -Rv /home/계정/www

9. 최종 전환 작업

9-1. 기존 서버 점검 모드 또는 쓰기 중지

게시글/댓글/회원가입/파일업로드가 발생하지 않도록 잠시 점검 모드로 전환한다.

9-2. 최종 DB 백업

mysqldump -u root -p \
  --single-transaction \
  --routines \
  --triggers \
  --events \
  --default-character-set=utf8mb4 \
  DB_NAME > /home/계정/migration-backup/DB_NAME-final-$(date +%F-%H%M).sql

9-3. 최종 파일 동기화

기존 서버에서 실행:

rsync -avz --delete \
  -e "ssh -i /path/to/key.pem" \
  /home/계정/www/ ec2-user@NEW_IP:/tmp/www-final/

새 서버에서 실행:

sudo rsync -av --delete /tmp/www-final/ /home/계정/www/
sudo chown -R nginx:nginx /home/계정/www
sudo chmod -R 707 /home/계정/www/data
sudo rm -rf /home/계정/www/data/cache/* /home/계정/www/data/session/*

9-4. 최종 DB 복원

mysql -u DB_USER -p DB_NAME < /tmp/DB_NAME-final-YYYY-MM-DD-HHMM.sql

9-5. Static IP 또는 DNS 전환

  • Lightsail Static IP 사용 시: 기존 인스턴스에서 Static IP 분리 → 새 인스턴스에 연결
  • DNS A 레코드 사용 시: A 레코드를 새 서버 IP로 변경

9-6. HTTPS 발급/갱신

sudo dnf install -y certbot python3-certbot-nginx
sudo certbot --nginx -d DOMAIN -d www.DOMAIN
sudo certbot renew --dry-run

10. 롤백 계획

전환 후 최소 1~2주 동안 기존 AL2 서버를 삭제하지 않는다.

문제 발생 시 롤백

  1. 새 서버에서 장애 확인.
  2. DNS A 레코드를 기존 OLD_IP로 되돌리거나 Static IP를 기존 인스턴스에 재연결.
  3. 기존 서버의 nginx/php-fpm/mariadb 상태 확인.
  4. 장애 원인 분석 후 새 서버에서 재테스트.
sudo systemctl status nginx php-fpm mariadb --no-pager
sudo tail -100 /var/log/nginx/error.log

12. 참고 메모

  • AL2023은 AL2와 패키지 구성이 다르므로 기존 패키지 목록을 그대로 복제하지 말고 필요한 구성요소를 새로 설치한다.
  • 아미나빌더/그누보드 환경은 data 디렉터리 권한, PHP 확장, DB 문자셋, nginx 보안 차단 설정이 중요하다.
  • nginx는 Apache의 .htaccess를 사용하지 않으므로 업로드 디렉터리 PHP 실행 차단을 nginx 설정에 직접 넣어야 한다.
  • PHP 8.0.30에서 운영 중이므로 새 서버의 PHP가 8.1/8.2 이상일 경우 Deprecated/Warning/치명 오류가 발생할 수 있다. 반드시 사전 테스트한다.
